# UX Principles

## Product experience goals

- Confidence: users trust totals, due logic, and projections.
- Clarity: each pay cycle clearly answers what to pay now.
- Control: users can adjust amounts, statuses, and trade-offs quickly.
- Calmness: interface avoids clutter and financial anxiety spikes.
- Continuity: every screen reinforces the same domain language.

## Interaction heuristics

- Show derived values near editable source values.
- Use progressive disclosure for advanced debt and projection settings.
- Keep destructive actions explicit and reversible when practical.
- Use plain labels over jargon unless the term is domain-critical.
- Surface consequences before commit for budget-impacting changes.

## Feedback patterns

- Immediate inline validation for invalid amounts and dates.
- Save states should be explicit: saving, saved, failed.
- Empty states must include next-step guidance.
- Warning states should explain impact, not only report an error.

## Information density

- Prioritize quick-scan summaries in page headers.
- Keep detail panes and tables available without hiding critical totals.
- Use collapsible sections for secondary information.

## Trust and explainability

- Every major computed total should have a short explanation.
- Calculation assumptions should be visible in context.
- Changes to pay-cycle outcomes should be traceable to inputs.
