# Information Architecture

## Primary navigation

Top-level sections:

1. Overview
2. Income
3. Accounts
4. Expenses
5. Debts
6. Payments

## Page responsibilities

Overview:

- High-level snapshot of income, commitments, and surplus or deficit.
- Category breakdown, debt progress, and budget flow visuals.

Income:

- Create and manage income sources and target accounts.
- Show normalized totals across fortnightly, monthly, annual views.

Accounts:

- Manage account definitions and account-level cashflow summaries.
- Support future transaction reconciliation entry points.

Expenses:

- Manage recurring and ad-hoc expenses with filters.
- Show normalized amounts and due pattern details.

Debts:

- Manage debt-specific fields and payoff projections.
- Highlight zero-interest debts and progress.

Payments:

- Execute pay-cycle planning checklist and transfer decisions.
- Capture actuals and cycle item statuses.

## Global layout model

Header:

- Household selector (future-ready), cycle date context, theme switch.

Main content:

- Page title and cycle-aware summary strip.
- Primary working surface (table, list, chart, or checklist).

Side panel or drawer:

- Detail editing and contextual explanation where needed.

## Cross-page utilities

- Frequency view toggle: fortnightly, monthly, annual.
- Global search for expenses and debts.
- Consistent account and category chips.
