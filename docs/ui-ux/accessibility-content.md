# Accessibility and Content Standards

## Accessibility baseline

- Meet WCAG 2.1 AA contrast targets.
- Ensure full keyboard navigation for all controls.
- Provide visible focus states across themes.
- Label all form fields and status actions clearly.
- Announce save and validation outcomes for assistive technologies.

## Financial clarity standards

- Show currency symbols consistently.
- Use consistent rounding and decimal display rules.
- Explain projected values as estimates when assumptions apply.

## Terminology standards

Use consistent terms across the product:

- Pay cycle
- Must-pay
- Smoothing contribution
- Surplus or deficit
- Transfer recommendation
- Debt-free by

Avoid synonyms that increase ambiguity for users.

## Microcopy style

- Prefer short, direct sentence structure.
- Put the important number first when possible.
- Use neutral tone for deficits and warning states.
- For errors, explain next action clearly.

## Error message pattern

- What happened
- Why it matters
- What to do next

Example:

- Unable to save this expense.
- The due pattern is incomplete.
- Add a due day or choose a recurring pattern, then save again.

## Empty state pattern

- Clarify what is missing.
- Explain why this page is useful.
- Provide one clear next action button.
